# Feedback Tool & AI Agent Pipeline

An internal, full-stack feedback-to-code pipeline. It combines a frontend feedback widget (cloning Userback features) with a backend orchestration server and specialized AI agents to automatically transform user feedback into code fixes and pull requests.

## 🚀 The Workflow

1.  **Capture**: Users use the **Feedback Widget** to select a UI area, snap a screenshot, and add a description.
2.  **Save & Analyze**: The **Backend Server** saves the feedback and immediately triggers the **Groq Vision Agent**.
3.  **Refine**: The **Groq Vision Agent** (Llama 4 Scout) analyzes the screenshot and context to generate a precise technical prompt for coding.
4.  **Execute**: The **Jules Agent** receives the prompt, executes the necessary code changes in the target repository, and opens a Pull Request.

## 🛠️ Frontend Integration

To integrate the feedback widget into any project, include the CSS and JS files and configure your backend endpoint.

### 1. Include the CSS
Add this to your `<head>`:
```html
<link rel="stylesheet" href="path/to/feedback-widget.css">
```

### 2. Configure the Widget
Set the global configuration before loading the JS:
```html
<script>
window.FEEDBACK_WIDGET_CONFIG = {
    // The endpoint where the widget will POST/GET data.
    endpoint: 'http://localhost:12345',
};
</script>
```

### 3. Include the JavaScript
Add this at the end of your `<body>`:
```html
<script src="path/to/feedback-widget.js"></script>
```

### Example Usage

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My App</title>
    <!-- Include the widget CSS -->
    <link rel="stylesheet" href="feedback-widget.css">
</head>
<body>
    <h1>Welcome to My App</h1>
    <p>Some content here...</p>

    <!-- Configure Backend Endpoint -->
    <script>
    window.FEEDBACK_WIDGET_CONFIG = {
        endpoint: 'http://localhost:12345'
    };
    </script>

    <!-- Include the widget JS -->
    <script src="feedback-widget.js"></script>
</body>
</html>
```

## 🏗️ Backend Server (`server.js`)

The Express server handles file storage and agent orchestration.

### Setup
```bash
cd feedback-tool
npm install
node server.js
```

### API Endpoints
-   `GET /api/jules/sources`: Retrieves available Jules repositories (with local caching).
-   `POST /api/feedback`: Saves screenshot/text and triggers Groq analysis.
-   `POST /api/send-to-jules`: Triggers the Jules Subagent to start a coding session and create a PR.

### Storage
Feedback is stored in the `feedbacks/` directory, organized by timestamp:
```text
feedbacks/
└── [timestamp]/
    ├── feedback.md      # User feedback text and references
    ├── screenshot.png   # Captured UI area
    └── jules_prompt.json # Generated by Groq for Jules
```

## 🤖 AI Agents (`/agents`)

### 1. Groq Vision OCR (`groq-vision-ocr`)
Uses **Meta Llama 4 Scout** via Groq's multimodal API. It cross-references the screenshot with the markdown context to understand the user's intent and UI state.
- **Output**: A technical `agent_prompt`.

### 2. Jules Subagent (`jules-subagent`)
A terminal client for the **Google Jules API**. It takes the refined prompt, clones the target repository, implements the fix, and optionally creates a PR.

---

## 🔑 Environment Configuration

Ensure the following environment variables are set in the respective agent directories or globally:

- **Groq Agent**: `GROQ_API_KEY` (in `agents/groq-vision-ocr/.env`)
- **Jules Agent**: `JULES_API_KEY` (in `agents/jules-subagent/.env`)

## ✨ Features
- **Visual Selection**: Users can draw a rectangle to select specific UI elements.
- **Automatic Context**: Screenshots are captured using `html2canvas` and paired with technical context.
- **End-to-End Automation**: From a bug report to a Pull Request without manual developer intervention.
- **Repository Integration**: Choose from connected GitHub repositories directly within the feedback modal.
