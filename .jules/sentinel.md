## 2024-05-20 - Missing Authentication on Vision Analysis Endpoint
**Vulnerability:** The `/api/vision/analyze` endpoint in `backend/src/routes.ts` was registered above the `AuthController.middleware` check. This made the endpoint completely unauthenticated, allowing any external actor to trigger potentially costly LLM vision analysis operations without restriction, risking abuse and API credit exhaustion.
**Learning:** In Express routers, the order of route registration determines the execution flow. Middleware applied via `router.use()` only affects routes registered *after* it. Grouping routes logically by authentication requirements is critical to ensure no endpoints accidentally bypass protection.
**Prevention:** Ensure strict grouping of routes in `routes.ts`. Always place protected routes explicitly below the `router.use('/api', ... AuthController.middleware ...)` block. During code reviews, explicitly check the position of newly added routes relative to authentication middleware.
