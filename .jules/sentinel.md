## 2025-02-23 - CRITICAL Fix unauthenticated arbitrary file read/write in VisionController
**Vulnerability:** The `/api/vision/analyze` endpoint in `VisionController` was exposed before the `AuthController.middleware` check in `routes.ts`, allowing unauthenticated access. Furthermore, it suffered from a path traversal vulnerability as it accepted absolute or relative paths for `mdFilePath`, `outputPath`, and local `imagePaths` via `req.body` without strict validation against the configured `config.feedbackDir`.
**Learning:** In an Express application, middleware execution order is critical. Registering endpoints before the authentication middleware accidentally makes them publicly accessible. Additionally, when handling file paths provided via API endpoints, relying on implicit assumptions (like paths being inside a tmp folder) without explicit path resolution and bounds checking (`path.relative` against a root directory) exposes the system to arbitrary file read/write vulnerabilities.
**Prevention:** 1. Ensure all protected API routes are registered strictly after the authentication middleware or explicitly wrapped with it. 2. For any endpoint that manipulates files based on user input, strictly enforce directory boundaries by using `path.resolve` to normalize the input path and then `path.relative(root, target)` to ensure the target is within the intended root directory, blocking cases where it starts with `..` or is an absolute path outside the root.
